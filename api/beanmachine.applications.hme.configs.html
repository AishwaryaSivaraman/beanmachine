class beanmachine.applications.hme.configs.InferConfig(n_iter: int, n_warmup: int, algorithm: str = 'NMC', n_chains: int = 2, queries: List[str] = <factory>, keep_warmup: bool = False, keep_logprob: bool = False, seed: int = 5123401)

Bases: object

A configuration class for MCMC posterior inference.

Parameters

  • n_iter – total number of posterior inference iterations

  • n_warmup – number of posterior inference iterations to discard as burn-in or warm-up period

  • algorithm – MCMC posterior inference method, defaults to ‘NMC’

  • n_chains – number of independent Markov chains running in parallel, defaults to 2

  • queries – a set of model parameters to store posterior MCMC to be returned to user

  • keep_warmup – flag of whether to keep the first n_warmup posterior samples, defaults to False

  • keep_logprob – flag of whether to keep track of log-likelihood along MCMC inference, defaults to False

  • seed – an integer number that controls the pattern of randomness, defaults to 5123401

class beanmachine.applications.hme.configs.MixtureConfig(use_null_mixture: bool = True, use_bimodal_alternative: bool = False, use_asymmetric_modes: bool = False, use_partial_asymmetric_modes: bool = False, null_prob_regression: beanmachine.applications.hme.configs.RegressionConfig = <factory>, sign_prob_regression: beanmachine.applications.hme.configs.RegressionConfig = <factory>)

Bases: object

A configuration class for mixture components.

Parameters

  • use_null_mixture – flag of whether the outcome is a mixture of null (0 effect) and alternative (non-zero effect), defaults to True

  • use_bimodal_alternative – flag of whether to consider H1 model as a mixture of positive (H1+) and negative (H1-) effects, defaults to False

  • use_asymmetric_modes – flag of whether H1+ and H1- are asymmetric around zero, defaults to False

  • use_partial_asymmetric_modes – flag of whether H1+ shares the same scale parameters of the random effects with H1-, defaults to False

  • null_prob_regression – model configurations for null mixture probability parameter (also known as movability)

  • sign_prob_regression – model configurations for bimodal sign probability parameter

A configuration class for regression model specifications.

Parameters

  • outcome – column name of outcome, defaults to ‘y’

  • stderr – column name of standard deviation of afore-mentioned distribution family, defaults to None

  • formula – statistical formula establishing the relationship between outcome and covariates, defaults to ‘1’

  • link – link function linking the systematic component (mixed effects) and outcome y, defaults to ‘identity’
