class beanmachine.ppl.experimental.vi.optim.MultiOptimizer

Bases: object

Base class of optimizers that make use of higher-order derivatives. Higher-order optimizers generally use torch.autograd.grad() rather than torch.Tensor.backward(), and therefore require a different interface from usual optimizers. In this interface, the step() method inputs a loss tensor to be differentiated, and backpropagation is triggered one or more times inside the optimizer. Derived classes must implement step() to compute derivatives and update parameters in-place.
